# COC KP Helper 后续开发路线图

## 路线定位

- 产品形态：本地单机守秘人工具
- 近期目标：持续提升跑团效率，而不是切换到后端协作系统
- 技术边界：继续保持静态可直开架构（`index.html + src/app.js`）

## 当前已落地能力（截至本路线图）

- 案件、场景、线索、NPC、线索道具、场景描述的完整 CRUD
- 跑团模式中的线索状态切换与快速记录
- 线索道具工作流（筛选、收藏、插入、复制、使用计数）
- 健康检查 3.0（风险分级 + 跳转处理）
- 团后总结中心（建议稿生成 + 一键写回）

## 下一阶段建议（按优先级）

### 1. 团后总结 2.0（最高优先）

- 给团后总结增加“按场次范围生成”（最近 1 场 / 3 场 / 全部）
- 把“建议稿写回”拆成可编辑草稿框，允许落库前手改
- 增加“导出本次总结为 Markdown”

### 2. 线索推进看板

- 新增“关键线索推进矩阵”（线索 x 场景）
- 显示每条关键线索的获取路径覆盖度
- 支持一键补齐“备用来源”空缺

### 3. 规则速查 2.0

- 增加站内搜索
- 增加收藏和常用置顶
- 支持跑团模式内侧边快速打开

### 4. 数据可靠性增强

- 导入时增加“预检查报告”（字段缺失、版本不匹配、脏数据告警）
- 增加“只导出当前案件”选项
- 增加“自动快照备份”开关

## 验收节奏建议

- 每个版本保持“一个核心能力 + 一轮稳定性修复”
- 每次发布前固定执行：
  - `node --check src/app.js`
  - `node scripts/link-audit.js`
- 每次发布后至少做一轮手动回归：
  - 案件库 -> 工作台 -> 线索道具库 -> 跑团模式 -> 导入导出
