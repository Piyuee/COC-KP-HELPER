# COC KP Helper 使用手册

## 1. 产品简介

COC KP Helper 是一个面向《克苏鲁的呼唤》7版守秘人的单机前端工具。它适合在备团和跑团时使用，核心目标是帮助你：

- 管理案件与核心真相
- 维护场景、线索、NPC、线索道具
- 维护可直接朗读的场景描述备用稿
- 记录跑团过程中的关键事件
- 快速检索当前案件内容
- 用收藏与标签管理常用线索道具，并一键插入跑团记录草稿
- 将数据保存在本地浏览器中

当前版本是静态网页应用，不依赖后端服务。你可以直接打开 `index.html` 使用。

## 2. 打开方式

### 直接打开

双击仓库中的 [index.html](/Users/xukaizhe/Documents/Codex/COC-KP-HELPER/index.html) 即可。

### 本地静态服务

如果你希望以本地站点方式打开，可以在仓库目录运行：

```bash
python3 -m http.server 4173
```

然后访问：

```text
http://127.0.0.1:4173
```

## 3. 数据保存方式

本产品会把你的内容保存在浏览器的 `localStorage` 中。

这意味着：

- 同一浏览器、同一设备下，刷新页面后数据仍然存在
- 更换浏览器或清理浏览器站点数据后，内容会消失
- 建议定期使用“导出数据”保存备份

## 4. 顶部按钮说明

### 新建案件

点击后会跳转到“案件库”，你可以直接填写案件表单并创建新案件。

### 导出数据

会把当前所有案件、场景、线索、NPC、线索道具、规则速查和跑团记录导出为一个 JSON 文件（含 `schemaVersion: 2`）。

建议在以下场景使用：

- 开始大量编辑前
- 每次大改案件结构后
- 更换设备前

### 导入数据

选择之前导出的 JSON 文件后，应用会用导入内容替换当前本地数据。

### 恢复示例

会清空当前本地数据，并恢复仓库内置的示例案件。

## 5. 页面功能说明

## 5.1 概览

“概览”页会展示当前选中案件的总体状态，包括：

- 场次记录数
- 场景数量
- 已获得线索 / 总线索
- 待推进线索数
- 核心真相
- 玩家已知
- 玩家误判
- 下一步建议
- 当前场景
- 团后总结中心（本场关键事件、玩家认知建议稿、下次开场建议稿）

适合在开团前快速复盘。

你也可以直接在“概览”页使用团后总结中心：

- 一键把建议稿写入“玩家已知”
- 一键把建议稿写入“下一步建议”
- 一键把两项都写回案件
- 一键把总结保存为跑团记录

## 5.2 案件库

左侧是案件列表，右侧是案件编辑表单。

你可以在这里：

- 创建新案件
- 编辑已有案件
- 删除案件
- 打开某个案件进入工作台

### 建议优先填写的字段

- 案件标题
- 时代与地点
- 一句话钩子
- 核心真相
- 玩家已知
- 玩家误判
- 下一步建议

## 5.3 案件工作台

这是最核心的页面。

它包含 5 个标签：

- 总览
- 场景
- 线索
- NPC
- 线索道具

左侧是列表，右侧是编辑器。你可以边看现有内容，边新增或修改。

### 总览

用于维护案件级信息：

- 核心真相
- 玩家已知
- 玩家误判
- 下一步建议
- 当前场景

### 场景

用于维护场景卡。建议至少填写：

- 场景名
- 场景类型
- 场景简介
- 氛围描述
- 关键线索
- 补救线索

还可以通过勾选方式关联：

- NPC
- 线索
- 线索道具

### 线索

用于维护调查线索。建议填写：

- 线索名
- 类型
- 来源
- 状态
- 内容
- 备用来源
- 导向

推荐状态使用方式：

- `未获得`：玩家还没接触
- `进行中`：已经注意到，但未完整掌握
- `已获得`：已经拿到并能推动剧情

### NPC

用于维护人物。建议填写：

- 姓名
- 身份 / 职业
- 当前态度
- 公开信息
- 隐藏秘密
- 动机
- 可透露线索

### 线索道具

用于维护线索道具与展示材料。建议填写：

- 标题
- 类型
- 展示模板
- 正文内容
- 揭示时机
- 作用

当前版本会在“案件工作台”和“线索道具库”里直接显示线索道具正文预览，并支持不同模板：

- 通用纸张
- 报纸剪报
- 私人信件
- 档案记录
- 医疗报告

这样你在跑团前可以更快检查材料内容、展示顺序和视觉语气。

## 5.4 场景描述库

这是一个专门给 `提前写好的场景描述` 预留的页面。

你可以在这里维护：

- 标题
- 绑定场景
- 顺位
- 标签
- 使用说明
- 描述正文

推荐用途：

- 开场朗读稿
- 压迫感描写
- 搜查成功后的环境反馈
- 转场描述
- 梦境 / 幻觉 / SAN 冲击前的气氛文字

如果你把它绑定到了某个场景，那么在“跑团模式”里切到该场景时，也能直接看到这些备用描述。

当前版本还支持：

- 按场景自动分组查看
- 未绑定场景的描述单独归档
- 在同一场景分组内用“上移 / 下移”调整朗读顺序

## 5.5 跑团模式

用于开团时快速记录和查看。

当前功能包括：

- 查看当前场景
- 查看场景简介与氛围
- 查看当前场景绑定的备用描述
- 查看待推进线索
- 查看该场景的补救线索提示
- 快速切换当前场景
- 快速切换线索状态
- 快速插入“当前场景关联道具 + 收藏道具”到跑团记录草稿
- 快速追加跑团记录
- 删除历史记录

### 推荐使用方式

当玩家做出关键行动时，立刻记录一句话，例如：

- “玩家决定先施压法医”
- “玩家错过了编辑部的速记页”
- “玩家决定夜访六码头仓库”

这样你在下次开团前就能直接回看。

## 5.6 线索道具库

用于集中查看和调用当前案件的所有线索道具。

适合在跑团前检查：

- 是否已经为关键线索准备了展示材料
- 是否漏掉了揭示时机
- 哪些道具应该收藏为常用

当前版本支持这些工作流能力：

- 组合筛选：`关键词 + 模板 + 标签 + 只看收藏`
- 卡片快捷操作：`收藏/取消收藏`、`复制正文`、`插入跑团记录`、`标记已使用`
- 使用统计：自动记录 `使用次数` 和 `最近使用时间`
- 快速插入文本：优先插入你在“线索道具”编辑器里填写的“快速插入文本”

## 5.7 案件关系板

在“案件工作台”的总览区域，当前版本会显示一个可点击、可直接修改关系的“案件关系板”。

它会把 `场景 / 线索 / NPC / 线索道具` 放在同一个关系视图里，帮助你快速检查：

- 当前聚焦节点关联了哪些内容
- 哪些场景已经放入调查推进点
- 哪些线索或道具还没有落到具体场景
- 当前信息是否过于集中在某一条路径上

推荐做法：

1. 先建立场景、线索、NPC 和线索道具
2. 在场景编辑器里勾选关联线索、NPC 和线索道具
3. 回到总览，在关系板左侧切换不同节点
4. 在右侧查看关联内容，并直接调整它的挂接关系
5. 如果需要更完整编辑，再用“跳到对应编辑项”继续补全

## 5.8 健康检查 3.0

当前版本的健康检查已经不只看“有没有线索”，还会额外提示这些结构问题：

- 只有一条关键线索
- 关键线索缺少补救来源
- 关键线索没有挂到场景
- 关键线索没有写导向
- 多条关键线索过度集中在同一场景
- NPC 已写了信息，但没有挂到场景
- 线索道具已写了内容，但没有挂到场景
- 场景既没有线索说明，也没有关联线索
- 关键线索仅出现在一个场景且没有备用来源
- 关键线索所在场景没有任何线索道具支撑
- 当前场景没有线索挂接、线索说明和补救路径
- 当前场景没有绑定场景描述

它更适合在备团阶段做一次结构检查，而不是等到跑团中途才发现调查会断线。

健康检查现在有三档风险级别：

- 高风险
- 中风险
- 建议优化

每条告警都可以点击“跳到对应编辑项”直接跳转到处理页面。

## 5.9 规则速查

当前提供的是按主题整理的 Keeper 高频参考：

- 检定判定
- 战斗速查
- 追逐流程
- 疯狂与 SAN
- 伤害与状态

它更适合你在跑团中快速决策，而不是替代完整规则书。

你现在也可以直接在这个页面里维护自己的速查内容：

1. 进入 `规则速查`
2. 点击右上角 `编辑规则速查`
3. 直接新增或修改分组
4. 直接新增、编辑或删除词条
5. 退出编辑后，页面会继续保持可点击跳转的速查模式

所有改动都会自动保存到本地，也会跟随 `JSON 导出 / 导入` 一起带走。

## 6. 推荐工作流

下面是一套比较顺手的使用顺序。

### 备团阶段

1. 新建案件
2. 在“总览”里写下核心真相、玩家已知和误判
3. 在“场景”里建立调查会经过的地点和事件
4. 在“线索”里录入关键线索与补救线索
5. 在“NPC”里补充人物秘密和动机
6. 在“线索道具”里补充展示材料
7. 在“总览”里指定当前场景

### 跑团阶段

1. 打开“跑团模式”
2. 根据当前场景查看氛围与补救线索
3. 每出现关键选择就写一条运行记录
4. 如果玩家刚拿到或错过线索，可以直接在跑团模式里切换线索状态
5. 玩家获得线索后，也可以回到“线索”标签里做更详细维护

### 团后整理

1. 回到“总览”更新玩家已知与误判
2. 修改下一步建议
3. 导出 JSON 作为备份

## 7. 导入导出建议

推荐的备份习惯：

- 每次大改前导出一次
- 每场团结束后导出一次
- 在尝试新结构前先导出

导入时请注意：

- 导入会覆盖当前本地数据
- 如果你不确定，先导出当前数据再导入

## 8. 当前版本适合做什么

当前版本已经适合：

- 单人备团
- 构建一个 one-shot 或短团案件
- 记录场景、线索、NPC、线索道具
- 在跑团时做简明事件记录

- 复杂权限系统

## 11. 常见问题

### 刷新后数据还在吗？

在同一浏览器里，通常还在，因为会保存在本地浏览器存储中。

### 为什么换浏览器后看不到数据？

因为数据目前保存在本地浏览器，不会自动同步到别的浏览器。

### 最安全的备份方式是什么？

定期点击“导出数据”，保存 JSON 文件。

### 我误点了“恢复示例”怎么办？

如果此前没有导出备份，当前版本无法恢复被覆盖的数据。
